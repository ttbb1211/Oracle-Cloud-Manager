# Oracle 管理面板

这是一个基于原版项目裁剪后的 **Oracle 专用私有管理面板**。

当前版本不再面向多云 / DNS 通用场景，而是聚焦 Oracle Cloud Infrastructure（OCI）的日常管理，保留常用实例操作能力，同时去掉 AWS、DNS、自动建网等当前不需要的部分。

## 当前版本定位

适合：
- 私有自用
- 多区域 Oracle 账户管理
- 常用实例运维操作
- 通过 Docker + 反代方式部署在自己的服务器上

不适合：
- 公共 SaaS
- 多租户平台
- 需要完整内建用户系统与细粒度权限控制的正式商用后台

## 当前版本保留功能

### Oracle 账户管理
- 添加 Oracle 账户
- 读取与保存 OCI 配置
- 支持同一套凭据按不同 `region` 拆成多条账户记录管理

### 实例管理
- 查看实例列表
- 查看实例基础信息
- 启动 / 停止 / 重启实例
- 创建实例
- 删除实例
- 修改实例配置
- 切换公网 IP
- 添加 IPv6
- 放通防火墙

### 引导卷管理
- 查看引导卷列表
- 删除引导卷

### 其它
- Telegram 设置页面
- Docker 部署

## 当前版本已去掉的内容

- AWS 相关能力与入口
- DNS 管理相关能力与页面
- 自动建网络相关入口
- 任务队列导航入口
- 与当前版本不匹配的部分仪表盘统计卡片

## 技术栈

- 前端：Vue 3 + Vite
- 后端：Node.js + Express
- 数据存储：lowdb

## 项目结构

```text
backend/    后端服务与数据
frontend/   前端项目
```

## 本地开发

安装依赖：

```bash
npm install
cd frontend && npm install
```

启动后端：

```bash
npm run server
```

启动前端：

```bash
cd frontend
npm run dev
```

## Docker 部署

快速启动：

```bash
docker compose up -d --build
```

当前默认通过 Docker 启动后端与前端静态资源服务。

## 端口与反代建议

建议仅让容器监听本机回环地址，再通过 Nginx / 宝塔反代对外提供访问。

当前推荐的映射方式：

```yaml
ports:
  - "127.0.0.1:3001:3001"
```

推荐反代后再额外启用：
- HTTPS
- Basic Auth / 宝塔密码访问

## 默认访问地址

本机访问：

```text
http://127.0.0.1:3001
```

如需公网访问，请使用你自己的反代域名，不建议直接开放 `IP:3001`。

## 数据说明

运行过程中产生的数据默认保存在：

```text
backend/data
```

该目录已加入 `.gitignore`，不建议提交到 Git。

## 安全提醒

当前版本已经比原版更收敛，但它本质上仍然是一个高权限 Oracle 管理后台。

建议至少做到：
- 只部署在自己的服务器上
- 使用自己的反代与 HTTPS
- 启用访问密码保护
- 不直接开放 `IP:3001`

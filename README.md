# Oracle Cloud Manager

这是一个基于原版项目裁剪并重构后的 **Oracle 专用私有管理面板**。

当前版本不再面向多云 / DNS 通用场景，而是聚焦 Oracle Cloud Infrastructure（OCI）的日常管理，保留常用实例操作能力，并围绕 **多区号账户管理** 做了专门收口。

---

## 当前版本定位

适合：
- 私有自用
- Oracle Cloud 多区域实例管理
- 同一套 OCI 凭据的多区号账户拆分管理
- 常用实例运维操作
- 通过 Docker + 反向代理方式部署在自己的服务器上

不适合：
- 公共 SaaS
- 多租户平台
- 需要完整内建用户系统与细粒度权限控制的正式商用后台

---

## 当前版本核心特性

### 1. Oracle 专用
当前版本已经移除多云平台定位，专注 Oracle Cloud：
- 去掉 AWS 相关能力与入口
- 去掉 DNS 管理相关能力与页面
- 去掉自动建网络相关入口
- 去掉任务队列导航入口
- 收口仪表盘与账户页，减少无效信息展示

### 2. 多区号账户管理
这是当前版本最重要的增强之一。

支持：
- 粘贴一套 OCI Config + 私钥
- 自动识别当前租户已订阅区号
- 勾选区号后批量生成多条 Oracle 账户
- 生成后的账户按 `租户名 / regionCode` 方式命名

例如：
- `putrinurifa / ap-tokyo-1`
- `putrinurifa / ap-singapore-1`
- `putrinurifa / us-sanjose-1`

同时做了这些收口：
- 母账户自动隐藏，避免与拆分后的多区账户重复显示
- 账户管理 / 云实例 / 仪表盘 三处页面统一去重逻辑
- HOME region 优先排序
- 历史老账户如果 `credentials.region` 为空，会尝试从 `configText` 中解析 `region=` 作为兼容回退

### 3. Oracle 常用运维操作
保留了当前版本最常用的 Oracle 管理能力：
- 查看实例列表
- 查看实例基础信息
- 启动 / 停止 / 重启实例
- 创建实例
- 删除实例
- 修改实例配置
- 切换公网 IP
- 添加 IPv6
- 放通防火墙
- 查看引导卷列表
- 删除引导卷

### 4. 连通性测试
账户页提供 **连通性测试** 按钮。

它会尝试读取当前账户对应 region 的实例列表，用于快速确认：
- OCI Config 可解析
- 私钥有效
- fingerprint 匹配
- 当前 region 的 Oracle API 可连通

> 注意：连通性测试不代表所有功能都已验证，只表示当前账户凭据与该 region 的基础读取链路正常。


## Docker 部署

快速启动：

```bash
docker compose up -d --build
```

当前容器默认监听：

```yaml
ports:
  - "127.0.0.1:3001:3001"
```

也就是说：
- 默认只监听本机回环地址
- 建议通过 Nginx / 宝塔反代后再对外提供访问

---

## 默认访问地址

本机访问：

```text
http://127.0.0.1:3001
```

如需公网访问，请使用你自己的反代域名，**不建议直接开放 `IP:3001`**。

---

## 反向代理与访问保护建议

推荐做法：
- 使用 Nginx / 宝塔反向代理
- 开启 HTTPS
- 启用 Basic Auth / 宝塔密码访问（宝塔 → 网站 → 对应站点 → 设置 → 访问限制 → 添加密码）
- 不直接暴露容器端口到公网

当前版本虽然已经比原版更收敛，但本质上仍然是一个 **高权限 Oracle 管理后台**，请至少保证第一层访问保护。

## 使用建议

推荐的使用方式：
1. 先录入一套有效的 OCI Config 与私钥
2. 点击 **识别多区号**
3. 勾选需要管理的 region
4. 批量生成多条 Oracle 账户
5. 在账户页先做 **连通性测试**
6. 再进入云实例页做实例运维

这样可以把“同一套凭据、多 region 管理”收敛到统一视图里，同时避免原始母账户重复出现。

---

## 当前已知限制

- 当前没有内建登录系统，建议依赖反代层做访问控制
- 连通性测试目前主要验证实例列表读取链路，不是全功能探测
- 某些历史老数据需要依赖 `configText` 回退解析 region
- 目前仍是私有自用定位，不是完整多租户后台产品
